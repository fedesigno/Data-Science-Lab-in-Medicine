Descrizione del problema: l'obiettivo di questo progetto quello di discriminare lesioni oncologiche eterogenee da lesioni omogenee, attraverso modelli di machine learning. In particolare, il dataset composto da 44 immagini gi segmentate: esse sono state precedentemente partizionate in regioni significative di interesse (ROI). Da tali immagini verranno estratte delle caratteristiche anatomiche delle lesioni, attraverso le quali sar succesivamente possibile allenare i modelli predittivi. Per classificare tali lesioni, verranno utilizzati alcuni modelli di machine learning supervisionati e non-supervisionati.

3. Features extraction¶

In medicina, una tecnica molto utilizzata e semplice per estrarre features dalle immagini quella che si basa sul dominio di conoscenza del medico. Quindi, spesso, si tratta di estrarre delle caratteristiche anatomiche dalle immagini, basandosi su biomarcatori noti a priori. Le features morfologiche pi utilizzate in questo approccio sono: il Metabolic Target Volume (MTV), la supericie, la Spherical Disporportion, la sfericit ed il rapporto superficie-volume.

Quindi, dalle immagini precedentemente visualizzate, sono state estratte le seguenti features:

  • VOLUME
  • SURFACE
  • SPHERICAL DISPROPORTION
  • SPHERICITY
  • SURFACE-to-VOLUME RATIO

La funzione compute_surface() prende in input l'immagine della lesione (gi trasformata in array) e la dimensione dei pixels e ne calcola la superficie. La funzione create_data(), dato il percorso delle immagini, crea un vettore features contenente le features precedentemente descritte ed un vettore di 0 o 1 (omogenea-0 oppure eterogenea-1).

def compute_surface(M, pix_dim):
    x__size = M.shape[0]
    y__size = M.shape[1]
    z__size = M.shape[2]
    dx = pix_dim[0]
    dy = pix_dim[1]
    dz = pix_dim[2]

    surface = 0;
    N = np.zeros((x__size,y__size,z__size))

    for k in range(z__size):
        for j in range(y__size):
            
            if M[1,j,k]>0:
                surface = surface+(dz*dy)
                N[1,j,k] = 1
                
            if M[x__size-1,j,k]>0:
                surface = surface+(dz*dy)
                N[x__size-1,j,k] = 1


    for k in range(z__size):
        for i in range(x__size):
            
            if M[i,1,k]>0:
                surface = surface+(dz*dx)
                N[i,1,k] = 1
            
            if M[i,y__size-1,k]>0:
                surface = surface+(dz*dx)
                N[i,y__size-1,k] = 1


    for j in range(y__size):
        for i in range(x__size):
            if M[i,j,1] > 0:
                surface = surface+(dy*dx)
                N[i,j,1] = 1
            
            if M[i,j,z__size-1] > 0:
                surface = surface+(dy*dx)
                N[i,j,z__size-1] = 1


    for i in range(x__size):
        for j in range(y__size):
            for k in range(z__size):
                
                if k > 1 and M[i,j,k] > 0 and M[i,j,k-1] == 0:
                    surface = surface+(dx*dy)
                    N[i,j,k] = 1
                if k < z__size-1 and M[i,j,k] > 0 and M[i,j,k+1] == 0:
                    surface = surface+(dx*dy)
                    N[i,j,k] = 1
                
                
                if i > 1 and M[i,j,k] > 0 and M[i-1,j,k] == 0:
                    surface = surface+(dz*dy)
                    N[i,j,k] = 1
                if i < x__size-1 and M[i,j,k] > 0 and M[i+1,j,k] == 0:
                    surface = surface+(dz*dy)
                    N[i,j,k] = 1
                
                
                if j > 1 and M[i,j,k] > 0 and M[i,j-1,k] == 0:
                    surface = surface+(dz*dx)
                    N[i,j,k] = 1
                if j < y__size-1 and M[i,j,k] > 0 and M[i,j+1,k] == 0:
                    surface = surface+(dz*dx)
                    N[i,j,k] = 1
    
    return surface, N

def create_data(path_type, y):
    
    data = []
    features = []
    files = os.listdir(path_type)
    
    for i in range(len(files)):
        img = nib.load(path_type+files[i])

        # Volume Lesione
        img_values = img.get_fdata()


        pix_dim = img.header['pixdim'][1:4]/10
        dim = img.header['dim'][1:4]
        
        
        #VOLUME
        lesion_voxels = np.where(img_values>0,1,0).sum()
        voxel_volume = (pix_dim).prod()
        lesion_volume = lesion_voxels*voxel_volume
        
        #SURFACE
        surf = compute_surface(img_values, pix_dim)
        
        lesion_area = surf[0]
        
        #SPHERICAL DISPROPORTION
        R_equiv = (lesion_volume*3/(4*math.pi))**(1/3);
        spherical_disproportion = lesion_area/(4*math.pi*R_equiv**2)
        
        #SPHERICITY
        sphericity = (math.pi**(1/3)*6*lesion_volume**(2/3))/lesion_area
        
        #SURFACE-to-VOLUME RATIO
        surfacevolume_ratio = lesion_area/lesion_volume
             
        data.append(img_values)
        features.append([lesion_volume, lesion_area, spherical_disproportion, 
                   sphericity, surfacevolume_ratio])

    #data = np.vstack(data)
    label = [y]*len(data)
    
    
    return features, label, data

4. Modelli di classificazione¶

Per classificare l'eterogeneit o l'omogeneit delle lesioni, sono stai utilizzati diversi modelli supervisionati e non-supervisionati. Per testare le capacit predittive dei modelli, stata utilizzata (per tutti i modelli) una Stratified K-fold cross validation, fissando il random seed a 42 e il numero di fold k=6. Attraverso questo procedimento stato possibile testare l'effettiva capacit dei modelli, indipendentemente dal test set utilizzato. Per ciascun modello sono stati riportati i valori di precision-recall-f1score per entrambe le classi. Inoltre, vengono riportate la accuratezze medie con i rispettivi massimi e minimi.

Il layer finale di output, ha come funzione di attivazione softmax. Come funzione di loss stata utlizzata una binary crossentropy e come funzione di ottimizzazione adam. 5. Risultati¶

Osservando le prestazioni ottenute dai modelli in k-fold cross validation, possibile affermare che il modello migliore risulta essere il Support Vector Classifier con circa il 80% di accuratezza media. Da notare, per, che tale valore ha un intervallo di variazione di circa il 14% (min: 71.43% - max: 85.71%): questo ci suggerisce che il modello molto sensibile ai dati di test che gli vengono forniti. In conclusione, possibile affermare che il modello non sar in grado di pervedere con ottima accuratezza il tipo di lesione: la ragione potrebbe essere legata al ristretto numero di immagini (44) utilizzate per allenare i modelli. Aumentando il numero di immagini si potrebbero ottenere risultati migliori e pi convincenti sia sulla SVC che su altri modelli.
